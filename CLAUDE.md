## Skill routing — read the matching skill BEFORE doing the work

Skills live in `.claude/skills/`. They are committed to this repo so every
environment (local, cloud session, teammate) has them. Match the task to the
skill and read its SKILL.md first:

### Engagement structure (always applies to client work)
- **Any 3D Process work, filing, naming, output codes, gate reviews** →
  `.claude/skills/jp-3d-core/SKILL.md`
- **Discover phase: audience teardown, competitor analysis, killer questions,
  discovery pack/synthesis, Gate 1** →
  `.claude/skills/jp-3d-discover/SKILL.md` (with jp-3d-core for filing)

### Writing
- **Writing NEW site copy** (headlines, hero, CTAs, page messaging) →
  `.claude/skills/copywriting/SKILL.md`
- **Editing/polishing EXISTING copy** the client supplied →
  `.claude/skills/copy-editing/SKILL.md`
- **ALL prose output, always** — before finalising any client-facing text,
  apply `.claude/skills/stop-slop/SKILL.md` to strip AI writing tells.

### Design & build
- **Landing pages, portfolios, redesigns — setting design direction** →
  `.claude/skills/taste-skill/SKILL.md` first
- **Building components/pages, typography, layout, avoiding generic AI look** →
  `.claude/skills/frontend-design/SKILL.md`
- **Concrete design-system picks: styles, palettes, font pairings, charts** →
  `.claude/skills/ui-ux-pro-max/SKILL.md`
- **Any animation, transitions, hover states, micro-interactions (build or
  audit)** → `.claude/skills/design-motion-principles/SKILL.md`

### Post-launch
- **SEO audit / rankings / indexing / core web vitals** →
  `.claude/skills/seo-audit/SKILL.md`

## Working rules

- Plan mode before multi-file changes; show the plan before executing.
- Verify with evidence: paste build/test output, don't just say "done".
- Commit in small, revertable chunks with clear messages.
- Client-facing copy is never finalised without a stop-slop pass.
